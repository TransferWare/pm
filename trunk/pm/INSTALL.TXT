1) Create an application owner

   a) change to install directory

   b) SQL> connect sys@<repository database>
      SQL> @ creusr <application owner> <default tablespace> <temporary tablespace>
      SQL> @ pm.grt <application owner>

   c) SQL> connect <application owner>@<repository database>
      SQL> @ pm_install.sql
      SQL> @ pm_build.sql

      During this step you will be prompted for several variables:
      i)   userid     - The connect string of the application owner
      ii)  pm_home    - The top of the PM distribution
      iii) pm_frm_bin - The directory to install forms executables into
                        Should be a subdirectory to &&pm_home.
      iv)  pm_rpt_bin - The directory to install reports executables into.
                        Should be equal to &&pm_frm_bin.
      v)   frmgen     - The forms compiler (ifcmp60/ifcmp90)
      vi)  rptgen     - The reports compiler (rwcon60/rwconverter)

2) Installation of Total Performance Index (optional)

   a) SQL> connect sys@<database to monitor>
   b) SQL> @ pm_tpi.sql

3) Installation of interface software

   The Performance Monitor creates interface views based on Oracle dictionary
   views. The view names start with pm$. These views hide differences between
   Oracle versions.

   a) Create interface account (on different database)

      SQL> connect sys@<database to monitor>
      SQL> @ creusr PM_LINK <default tablespace> <temporary tablespace>
      SQL> @ pm_remote.grt <interface account>
      SQL> connect <interface account>@<database to monitor>
      SQL> @ pm_remote.sql

   b) Create database link (monitored database)

      SQL> connect <application owner>@<repository database>
      SQL> @ credblnk <database link> <interface account> <interface account password> <tns service>

      Test the link by:

      SQL> select * from user_views@<database link> where view_name like 'pm$%';

4) Collect information

   Collect information for database denoted by database link MYDB:

   SQL> execute pm.collect('MYDB')

   or collect information about the current database by:

   SQL> execute pm.collect(NULL)

5) Cleanup

   Clean up 14 days of history for database MYDB:

   SQL> execute pm.cleanup('MYDB', 14)

   or clean up for the current database:

   SQL> execute pm.cleanup(NULL, 14)
